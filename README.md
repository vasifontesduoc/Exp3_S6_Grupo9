# Exp3_S6_Grupo9
# Componentes PL/SQL en base de datos
La actividad corresponde a la experiencia 3 con semana número 6, donde se desarrollaron procedimientos almacenados en PL/SQL para detectar departamentos que no han pagado sus gastos comunes.
El sistema revisa los pagos de los períodos anteriores, identifica a los deudores y aplica automáticamente la multa correspondiente según las reglas del negocio. Además, genera un registro con la información de los departamentos en mora y actualiza la multa en la tabla de gastos comunes.
Se utilizaron procedimientos con parámetros de entrada y salida, manejo de excepciones para evitar errores por datos duplicados, y comentarios en todo el código para facilitar su comprensión.
